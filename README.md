# 可视化停车场管理系统

一个基于 Qt 6 的可视化停车场管理系统，支持管理员和普通用户角色，提供车辆管理、路径规划动画、计费服务等功能。

## 功能特性

### 管理员功能
- **车辆管理**：添加、删除车辆，支持自定义车牌号、车型、颜色和图片
- **车辆驶入/驶出**：支持手动顺序驶入、随机车位驶入和自动驶入模式
- **路径规划动画**：车辆沿道路直线行驶，自动规划从等待队列到目标车位的路径（含倒车入库动画）
- **计费服务**：支持首小时费率、额外小时费率、每日最高收费、免费时长等配置
- **车辆详情**：点击车辆可查看详细信息（停车位置、进入时间、停车时长、实时费用），每秒动态更新
- **信息修改**：支持在详情对话框中编辑车辆信息

### 普通用户功能
- **停车场查看**：浏览停车场实时状态（车位占用/空闲）
- **登录系统**：普通用户登录后可查看停车场信息

### 系统设置
- **停车场配置**：自定义停车场行数和列数（1-20 范围），自动重绘
- **计费规则设置**：灵活配置计费参数，持久化到 SQLite 数据库

## 技术架构

- **语言**：C++17
- **框架**：Qt 6.10.2
- **构建工具**：CMake 3.16+
- **图形视图**：QGraphicsView / QGraphicsScene（场景-视图分离架构）
- **动画系统**：QPropertyAnimation + QSequentialAnimationGroup
- **数据存储**：SQLite（Qt SQL 模块），密码 SHA-256 加盐哈希
- **测试框架**：Qt Test
- **设计模式**：接口抽象（IFeeService / IAuthService）、依赖注入、Result 模式、Hub 中枢（MainWindow）、观察者（信号槽）

## 编译运行

### 环境要求
- Qt 6.x（推荐 6.10.2）
- CMake 3.16+
- MinGW 或 MSVC 编译器

### 编译步骤

1. 打开 Qt Creator
2. 打开项目目录下的 `CMakeLists.txt`
3. 选择构建配置（Debug/Release）
4. 点击构建按钮

编译成功后，运行生成的可执行文件 `VisualParkingManagementSystem.exe`。

### 运行测试

```bash
cd build/Desktop_Qt_6_10_2_MinGW_64_bit-Debug
ctest
```

## 使用说明

### 登录系统
1. 打开应用程序
2. 点击菜单「登录」
3. 选择登录类型（管理员 / 普通用户）
4. 输入用户名和密码
   - 默认管理员：用户名 `stay`，密码 `123456`
   - 默认普通用户：用户名 `user`，密码 `123456`

### 管理员操作

#### 添加车辆
1. 点击菜单「管理」->「打开管理界面」
2. 输入车牌号、车型
3. 选择车辆颜色（可选）
4. 选择车辆图片（可选）
5. 点击「添加车辆」，车辆进入右侧等待队列

#### 车辆驶入
- **顺序模式**：点击「车辆进入(顺序)」，车辆从等待队列依次进入停车场
- **自动模式**：点击「车辆进入（auto）」，系统每 1.5 秒自动驶入一辆车，优先选择靠近入口的空位

#### 车辆驶出
- 点击「车辆驶出」，车辆按进入顺序依次驶出并播放驶离动画

#### 查看车辆详情
- 点击停车场内的任意车辆，弹出详情对话框
- 显示车辆信息、停车位置、进入时间、停车时长和当前费用（每秒实时更新）

### 系统设置
1. 点击菜单「停车场设置」
2. **停车场设置**：设置行数和列数（1-20），点击确认后自动重绘
3. **计费规则设置**：
   - 首小时费率：第一小时的收费标准（默认 5 元）
   - 额外小时费率：超过一小时后的收费标准（默认 3 元/小时）
   - 每日最高收费：单日最高收费上限（默认 50 元）
   - 免费时长：停车免费时长（默认 15 分钟）

## 计费规则

计费公式：
```
停车时长 ≤ 免费时长 → 费用 = 0
停车时长 ≤ 1 小时  → 费用 = 首小时费率
停车时长 > 1 小时  → 费用 = 首小时费率 + (小时数 - 1) × 额外小时费率
最终费用 = min(计算费用, 每日最高收费)
```

示例（默认规则）：
- 停车 10 分钟 → 0 元（免费时长内）
- 停车 30 分钟 → 5 元（首小时）
- 停车 2 小时 → 8 元（5 + 3）
- 停车 20 小时 → 50 元（触及日封顶）

## 路径规划

PathPlanner 模块生成车辆从等待队列到目标车位的完整行驶路径：

```
等待队列 → 右侧垂直车道 → 目标行道路 → 车位正前方 → 倒车入库
```

- 5 个控制点定义路径骨架，4 段直线，每段 20 个插值点（共 80 个路径点）
- 每步动画 40ms，总驶入时长约 3.2 秒
- 驶入末尾带有 -8° 旋转动画，模拟倒车入库
- 驶出动画 3 段直线，总时长约 0.8 秒
- 防抖机制（isCarEntering）防止动画冲突

## 设计模式

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| 接口抽象 | IFeeService / IAuthService | 面向接口编程，便于测试和替换实现 |
| 依赖注入 | MainWindow 构造接收 Service 指针 | 解耦模块，由 main.cpp 组装依赖 |
| Result 模式 | Result\<T\> | 统一的错误处理，替代异常 |
| Hub 中枢 | MainWindow | 所有模块通信的中枢 |
| 观察者 | 信号槽机制 | Qt 原生跨模块通信 |
| MVC 分离 | QGraphicsScene / View | Scene = Model，View = View |

## 数据存储

- **SQLite 数据库**（`vpms.db`）：存储用户账号（SHA-256 加盐哈希密码）和计费规则
- 首次启动时自动从旧版文本文件（`name_password.txt`、`fee_rule.txt`）迁移数据
- DatabaseManager 封装所有数据库操作，使用 Result\<T\> 返回结果
